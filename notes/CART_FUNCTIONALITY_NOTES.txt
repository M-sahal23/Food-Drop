                    FEATURES IMPLEMENTED

1. ADD TO CART FUNCTIONALITY
   ├─ Function: AddToCart(request, id)
   ├─ Location: foods/views.py
   ├─ How it works:
   │  • Retrieves the food item by ID
   │  • Creates or gets existing cart item
   │  • If new: adds with quantity=1 and total_price=food.price
   │  • If exists: increases quantity and updates total_price
   │  • Redirects to cart page
   └─ URL Pattern: path('cart/<int:id>', views.AddToCart, name='addtocart')


2. DELETE FROM CART FUNCTIONALITY
   ├─ Function: delete_from_cart(request, id)
   ├─ Location: foods/views.py
   ├─ How it works:
   │  • Gets the cart item by its ID
   │  • Deletes the item permanently
   │  • Redirects back to cart page
   │  • JavaScript confirmation dialog prevents accidental deletion
   └─ URL Pattern: path('cart/delete/<int:id>/', views.delete_from_cart, name='delete_from_cart')


3. UPDATE QUANTITY FUNCTIONALITY
   ├─ Function: update_quantity(request, id)
   ├─ Location: foods/views.py
   ├─ How it works:
   │  • Receives POST request with action parameter
   │  • If action='increase': quantity += 1
   │  • If action='decrease': quantity -= 1 (minimum 1)
   │  • Recalculates: total_price = quantity × food.price
   │  • Saves changes and redirects to cart
   └─ URL Pattern: path('cart/update-quantity/<int:id>/', views.update_quantity, name='update_quantity')


4. CART VIEW DISPLAY
   ├─ Function: cart_view(request)
   ├─ Location: foods/views.py
   ├─ How it works:
   │  • Retrieves all cart items from database
   │  • Calculates grand_total by summing all item total_prices
   │  • Passes cart_items and grand_total to template
   │  • Shows "Your cart is empty" if no items
   └─ Template: foods/cart.html

   ```
       cart_item.delete()
       return redirect('cart')
   ```


CHANGE 3: Added update_quantity function
   Purpose: Increase/decrease item quantity and recalculate price

   Key Code:
   ```
   def update_quantity(request, id):
       """Update the quantity of a food item in the cart"""
       if request.method == 'POST':
           action = request.POST.get('action')
           cart_item = get_object_or_404(Cart, id=id)
           
           if action == 'increase':
               cart_item.quantity += 1
           elif action == 'decrease' and cart_item.quantity > 1:
               cart_item.quantity -= 1
           
           cart_item.total_price = cart_item.quantity * cart_item.food.price
           cart_item.save()
       
       return redirect('cart')
   ```

DATABASE MODEL: Cart
─────────────────────────────────────────────────────────────────────────────
class Cart(models.Model):
    food = models.ForeignKey("FoodItems", related_name="fooditems", 
                             on_delete=models.CASCADE)
    quantity = models.IntegerField(default=1)
    total_price = models.IntegerField()


KEY CONCEPTS:

1. FOREIGN KEY RELATIONSHIP
   • Cart.food links to FoodItems model
   • Allows access to food details via item.food.name, item.food.price, etc.

2. AUTOMATIC CALCULATIONS
   • total_price is recalculated: quantity × food.price
   • grand_total in view: sum of all item.total_price values

3. SECURITY
   • {% csrf_token %} in forms prevents CSRF attacks
   • onclick="return confirm(...)" prevents accidental deletions
   • get_object_or_404 prevents accessing non-existent items

4. REDIRECT PATTERN
   • All POST operations redirect to 'cart' page
   • Prevents form resubmission issues
   • User always sees current cart state

HOW PRICE CALCULATION WORKS:
────────────────────────────

1. BASE PRICE: Food item price (e.g., Pizza = ₹300)

2. ADDITIONS:
   ├─ Size: +price (e.g., Large = +₹50)
   ├─ Sauce: +price × quantity (e.g., Extra Garlic = +₹10 per unit)
   └─ Toppings: +price each × quantity (e.g., Cheese = +₹20 per unit)

3. CALCULATION:
   Total per item = Base Price + (Size Price) + (Sauce Price × Sauce Qty) + 
                    (Sum of Topping Prices × Individual Topping Quantities)
   Final Total = Total per item × Quantity

4. EXAMPLE:
   Base Pizza: ₹300
   + Large Size: +₹50 (1×)
   + Garlic Sauce: +₹10 (2 units = 2×₹10 = ₹20)
   + Cheese Topping: +₹20 (1 unit = ₹20)
   + Pepperoni Topping: +₹15 (3 units = 3×₹15 = ₹45)
   = Item Price: ₹435
   × Quantity: 2
   = Final Total: ₹870

KEY TECHNICAL CONCEPTS:
───────────────────────

1. FOREIGNKEY vs MANYTOMANY:
   ├─ Size, Base, Sauce: ForeignKey (one per item)
   └─ Toppings: ManyToMany (multiple per item)

2. DECIMAL vs INTEGER:
   ├─ CustomizedCart uses DecimalField for accurate prices
   ├─ Regular Cart uses IntegerField (can be upgraded to Decimal)
   └─ Reason: Handles decimal prices (₹10.50, etc.)

3. PRICE CALCULATION METHOD:
   • calculate_total_price() on CustomizedCart model
   • Loops through all related objects
   • Sums prices and multiplies by quantity
   • More reliable than template calculations

4. FORM SUBMISSION:
   • POST request to save_customization view
   • CSRF token included for security
   • Extract form data: request.POST.get() and request.POST.getlist()
   • Create objects and save relationships

5. TEMPLATE LOOPS:
   • {% for topping in item.toppings.all %}
   • Accesses ManyToMany related objects
   • Can be empty if no toppings selected

   ✓ Less visual clutter

   ✓ Better mobile experience

   ✓ Simpler view logic

   ✓ Fewer form fields to process

HOW USERS DESELECT:
─────────────────────

Radio Fields (Size, Base, Sauce):
  • Click the "❌ No [Option]" radio button
  • This deselects the previously selected option
  • Empty value is sent to server

Checkboxes (Toppings):
  • Click the checkbox again to uncheck it
  • No need for "No Toppings" option
  • Just leave all unchecked
