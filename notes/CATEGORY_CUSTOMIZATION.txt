The Category-Based Customization System intelligently shows/hides customization 
options based on the food item's category. This provides:

✓ Cleaner, less overwhelming UI for users
✓ Context-relevant options (e.g., no "size" for a single-serving item)
✓ Improved user experience by reducing irrelevant choices
✓ Flexible system that adapts to new food categories

2.1 DETAILED CATEGORY RULES & RATIONALE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

───────────────────────────────────────────────────────────────────────────────
PIZZA
───────────────────────────────────────────────────────────────────────────────
Available Options: SIZE ✓ | BASE ✓ | SAUCE ✓ | TOPPINGS ✓

Rationale:
  • Pizzas come in multiple sizes (Regular, Medium, Large, Extra Large)
  • Multiple base options (Thin Crust, Pan, Stuffed Crust, etc.)
  • Various sauce choices (Tomato, White, BBQ, etc.)
  • Multiple toppings for personalization

Example Customization:
  Pizza Margherita + Large Size + Stuffed Crust + Tomato Sauce + Cheese + Mushrooms

───────────────────────────────────────────────────────────────────────────────
BURGER
───────────────────────────────────────────────────────────────────────────────
Available Options: BASE ✓ | SAUCE ✓ | TOPPINGS ✓ | SIZE ✗

Rationale:
  • Burgers are typically single-size items (no size variation)
  • Customers can choose bun type (base): Regular, Toasted, Multigrain, etc.
  • Sauce customization: Mayo, Mustard, BBQ, Sriracha, etc.
  • Toppings: Lettuce, Tomato, Onions, Cheese, Bacon, etc.

Example Customization:
  Cheese Burger + Toasted Bun + Mayo Sauce + Lettuce + Tomato

Why No Size?
  • Burgers are portioned items; size variation doesn't apply
  • Avoids confusing users with irrelevant "size" options

───────────────────────────────────────────────────────────────────────────────
BIRIYANI
───────────────────────────────────────────────────────────────────────────────
Available Options: TOPPINGS ✓ ONLY | BASE ✗ | SAUCE ✗ | SIZE ✗

Rationale:
  • Biryanis are complete, pre-made dishes with set portions
  • No base/size/sauce options - served as-is
  • Only toppings allowed for add-ons: Boiled Eggs, Raita, Pickles, etc.

Example Customization:
  Chicken Biryani + Extra Boiled Eggs + Raita

Why Minimal Options?
  • Biryani is a complex dish with balanced ingredients
  • Additional customization would disrupt the traditional recipe

───────────────────────────────────────────────────────────────────────────────
FRENCH FRIES
───────────────────────────────────────────────────────────────────────────────
Available Options: SIZE ✓ | SAUCE ✓ | TOPPINGS ✓ | BASE ✗

Rationale:
  • Multiple size options: Small, Medium, Large, Extra Large
  • Various sauce coatings: Cheese, BBQ, Peri Peri, etc.
  • Toppings: Bacon, Jalapeños, Onions, etc.
  • No "base" needed - fries are the base

Example Customization:
  French Fries Large + Peri Peri Sauce + Cheese + Jalapeños

───────────────────────────────────────────────────────────────────────────────
DESSERTS
───────────────────────────────────────────────────────────────────────────────
Available Options: TOPPINGS ✓ ONLY | BASE ✗ | SAUCE ✗ | SIZE ✗

Rationale:
  • Desserts are finished products (cakes, ice cream, pastries)
  • No base/size/sauce customization needed
  • Only optional toppings: Sprinkles, Chocolate, Nuts, Whipped Cream, etc.

Example Customization:
  Chocolate Cake + Extra Chocolate Sprinkles + Whipped Cream

───────────────────────────────────────────────────────────────────────────────
BEVERAGES
───────────────────────────────────────────────────────────────────────────────
Available Options: SIZE ✓ ONLY | BASE ✗ | SAUCE ✗ | TOPPINGS ✗

Rationale:
  • Beverages primarily vary by size: Small, Medium, Large
  • No base/sauce/toppings needed
  • Size affects price and portion

Example Customization:
  Cola Large

3.1 BACKEND: views.py - Customize_Food() Function
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Location: foods/views.py → Customize_Food(request, id=0)

Function Flow:
  1. Get food item from database using ID
  2. Determine food category
  3. Set show_* boolean flags based on category
  4. Only query DB for options that will be shown
  5. Pass flags and options to template

Code Structure:

  def Customize_Food(request, id=0):
      food = get_object_or_404(FoodItems, id=id)
      
      # Category-based conditional flags
      if food.category == "PIZZA":
          show_size = True
          show_base = True
          show_sauce = True
          show_toppings = True
      elif food.category == "BURGER":
          show_size = False
          show_base = True
          show_sauce = True
          show_toppings = True
      elif food.category == "BIRIYANI":
          show_size = False
          show_base = False
          show_sauce = False
          show_toppings = True
      # ... additional categories ...
      
      # Only query options that will be displayed
      size_opt = SizeChart.objects.all() if show_size else []
      base_opt = BaseType.objects.all() if show_base else []
      sauce_opt = Sauce.objects.all() if show_sauce else []
      toppings_opt = Toppings.objects.all() if show_toppings else []
      
      context = {
          'food': food,
          'show_size': show_size,
          'show_base': show_base,
          'show_sauce': show_sauce,
          'show_toppings': show_toppings,
          'food_category': food.category,
          'size_opt': size_opt,
          'base_opt': base_opt,
          'sauce_opt': sauce_opt,
          'toppings_opt': toppings_opt,
      }
      return render(request, 'foods/customize.html', context)

Context Variables Passed:
  • show_size: Boolean - Display SIZE section
  • show_base: Boolean - Display BASE section
  • show_sauce: Boolean - Display SAUCE section
  • show_toppings: Boolean - Display TOPPINGS section
  • food_category: String - Food's category (for display info)
  • size_opt, base_opt, sauce_opt, toppings_opt: Lists - Available options

Key Design Decisions:
  ✓ Database queries only for shown options (performance optimization)
  ✓ Clear, readable if/elif blocks for each category
  ✓ Centralized category logic makes updates easy

Location: templates/foods/customize.html

Template Structure:

1. Category Info Box:
   Shows which category is selected and what customizations are available
   
   {% if food_category == "PIZZA" %}
     <p>✓ You can customize: Size, Base, Sauce, Toppings</p>
   {% elif food_category == "BURGER" %}
     <p>✓ You can customize: Base, Sauce, Toppings (No Size option)</p>
   {% endif %}

2. Conditional Option Groups:
   Each customization section is wrapped with {% if show_* %} condition
   
   SIZE SECTION:
   {% if show_size %}
     <div class="option-group">
       <h5>📏 Select Size</h5>
       <!-- Radio buttons for sizes -->
     </div>
   {% endif %}
   
   BASE SECTION:
   {% if show_base %}
     <div class="option-group">
       <h5>🥄 Select Base</h5>
       <!-- Radio buttons for bases -->
     </div>
   {% endif %}
   
   SAUCE SECTION:
   {% if show_sauce %}
     <div class="option-group">
       <h5>🍅 Select Sauce</h5>
       <!-- Radio buttons for sauces -->
     </div>
   {% endif %}
   
   TOPPINGS SECTION:
   {% if show_toppings %}
     <div class="option-group">
       <h5>🧅 Select Toppings (Multiple)</h5>
       <!-- Checkboxes for toppings -->
     </div>
   {% endif %}

Key Features:
  ✓ Dynamic rendering based on view context
  ✓ User-friendly category info display
  ✓ Responsive layout adjusts to shown sections
  ✓ Consistent styling across all options

4. USER EXPERIENCE FLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step-by-Step User Journey:

1. User browses food items list (home.html or allfoods.html)

2. User clicks "Customize" button on a food item
   URL: /customize/<food_id>/

3. Backend (Customize_Food view):
   ✓ Retrieves food item
   ✓ Checks food.category
   ✓ Sets show_* flags based on category rules
   ✓ Queries only relevant options
   ✓ Passes to template

4. Frontend (customize.html) renders:
   ✓ Shows category info: "📍 Category: Pizza"
   ✓ Shows relevant options based on category
   ✓ Hides irrelevant options
   ✓ User sees clean, organized form

5. User makes selections:
   ✓ For Pizza: Can select size, base, sauce, toppings
   ✓ For Burger: Can select base, sauce, toppings (NO size section visible)
   ✓ For Biryani: Can select toppings ONLY
   ✓ Etc.

6. User clicks "Add to Cart"
   Form submission to: /customize/save/<food_id>/

7. Backend (save_customization view):
   ✓ Receives POST data with selections
   ✓ Creates CustomizedCart entry with selections
   ✓ Calculates total price
   ✓ Redirects to cart view

8. User sees customized item in cart with breakdown:
   - Base Price: ₹300
   - Size: +₹50
   - Sauce: +₹10
   - Toppings: +₹30
   - Total: ₹390
