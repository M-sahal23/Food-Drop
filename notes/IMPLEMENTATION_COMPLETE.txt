TECHNICAL DETAILS:
═══════════════════════════════════════════════════════════════════════════

FORM INPUT NAMES:
  • Sauce: sauce_qty_<sauce_id>
  • Toppings: topping_qty_<topping_id>
  
  Example:
  - sauce_qty_1 (Garlic Sauce, ID=1)
  - topping_qty_5 (Cheese, ID=5)

DJANGO VIEW LOGIC:
  ```python
  # Read sauce quantity
  sauce_qty = int(request.POST.get(f'sauce_qty_{sauce_id}', 1))
  
  # Read topping quantities
  for topping_id in toppings_ids:
      topping_qty = int(request.POST.get(f'topping_qty_{topping_id}', 1))
      # Add topping 'topping_qty' times
      for _ in range(topping_qty):
          customized_item.toppings.add(topping)
  ```

PRICE CALCULATION:
  ```
  total = base_price
  + size_price (if selected)
  + (sauce_price × sauce_qty) (if selected)
  + sum(topping_price × topping_qty for each topping)
  = item_total
  × main_quantity
  = final_total
  ```

SUPPORT & TROUBLESHOOTING:
═════════════════════════════════════════════════════════════════════════════

Q: Quantity inputs not appearing?
A: Clear browser cache and refresh page.
   Check that customize.html was properly saved.

Q: Prices not calculating correctly?
A: Check that form field names match:
   - sauce_qty_<id>
   - topping_qty_<id>

Q: Toppings appearing multiple times in cart?
A: This is expected behavior! If user selects Cheese qty:3,
   it shows 3 separate lines. This is correct.

Q: Can customer reduce to 0 quantity?
A: Yes, with current implementation (min="0").
   Consider changing to min="1" if preferred.

Q: Mobile display issues?
A: Check that flex layout is supported.
   Test on actual mobile devices if possible.
