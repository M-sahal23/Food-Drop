================================================================================
                          WHAT DATA IS STORED
================================================================================

When user clicks "Buy Now" and places order, the following is stored:

1. UNIQUE ORDER ID
   Format: ORD_20260128143025_A7F2B3E1
   Components:
   - ORD_ (prefix)
   - 20260128143025 (timestamp YYYYMMDDHHMMSS)
   - A7F2B3E1 (8-char UUID hex)

2. CUSTOMER NAME
   Collected from modal input field
   Required field (cannot be empty)

3. ORDERED FOOD NAMES
   Stored as food items linked to order
   Each item has food name, quantity, price

4. QUANTITY
   Individual quantity for each food item
   Can be different for each item in order

5. TOTAL PRICE
   Calculated from all items
   Regular item total + Customized item total

6. ORDER DATE/TIME
   Auto-created timestamp
   Format: YYYY-MM-DD HH:MM:SS

7. ORDER STATUS
   Default: PENDING
   Can be updated to: PROCESSING, COMPLETED, CANCELLED
   Changeable from admin panel

8. CUSTOMIZATION DETAILS (if customized items)
   Stored as JSON string
   Contains: size, base, sauce, toppings selected

================================================================================
                        HOW THE FLOW WORKS
================================================================================

STEP-BY-STEP PROCESS:

1. USER ADDS ITEMS TO CART
   └─ Items stored in Cart or CustomizedCart tables
   └─ Cart icon shows item count

2. USER CLICKS "BUY NOW" BUTTON
   └─ Location: Shopping cart page
   └─ Button: Green with shopping bag icon
   └─ Action: Opens modal dialog

3. MODAL APPEARS
   ┌─────────────────────────────────────┐
   │ Complete Your Order                 │
   ├─────────────────────────────────────┤
   │ Customer Name: [________________]    │
   │ Order Summary:                      │
   │ Total Amount: ₹2,500.00             │
   ├─────────────────────────────────────┤
   │ [Cancel] [Confirm Purchase]         │
   └─────────────────────────────────────┘

4. USER ENTERS NAME
   └─ Name field is required
   └─ Validation on both client & server side

5. USER CLICKS "CONFIRM PURCHASE"
   └─ POST request sent to /foods/buy-now/
   └─ Customer name is submitted

6. SERVER PROCESSES ORDER
   ├─ Validate customer name exists
   ├─ Get all regular cart items
   ├─ Get all customized cart items
   ├─ Calculate total price
   ├─ Check cart not empty
   ├─ Create Order object
   │  └─ Auto-generates unique order_id
   │  └─ Sets status = PENDING
   │  └─ Records timestamp
   ├─ Create OrderItem for each regular item
   ├─ Create OrderItem for each customized item
   │  └─ Serializes customization as JSON
   ├─ Delete all cart items
   └─ Redirect to confirmation page

7. CONFIRMATION PAGE DISPLAYS
   ┌────────────────────────────────────┐
   │ ✅ Order Confirmed!                │
   ├────────────────────────────────────┤
   │ Order ID: ORD_20260128143025_A7F2  │
   │ Customer: John Doe                 │
   │ Status: PENDING                    │
   │ Date: 28-Jan-2026, 02:30 PM       │
   ├────────────────────────────────────┤
   │ Items:                             │
   │ • Pizza x2 - ₹600.00               │
   │ • Burger (Customized) x1 - ₹250    │
   ├────────────────────────────────────┤
   │ Total: ₹850.00                     │
   ├────────────────────────────────────┤
   │ [Continue Shopping] [Back to Home] │
   └────────────────────────────────────┘

   └─ Order record created

================================================================================
                       KEY FEATURES EXPLAINED
================================================================================

1. UNIQUE ORDER ID GENERATION
   
   Format: ORD_YYYYMMDDHHMMSS_XXXXXXXX
   
   Example Breakdown:
   ORD_20260128143025_A7F2B3E1
   ├─ ORD_ : Prefix (constant)
   ├─ 20260128143025 : Timestamp (2026-01-28 14:30:25)
   └─ A7F2B3E1 : 8-char UUID hex (random)
   
   Why this format:
   ✓ Human readable (contains date/time)
   ✓ Guaranteed unique (UUID ensures no duplicates)
   ✓ Sortable (timestamp allows chronological sort)
   ✓ Non-sequential (no pattern to guess)
   
   Generation Logic:
   ```python
   order_id = f"ORD_{datetime}.strftime('%Y%m%d%H%M%S')}_{uuid.uuid4().hex[:8].upper()}"
   ```

2. CART ITEM COUNT BADGE
   
   JavaScript Code:
   ```javascript
   function updateCartCount() {
       fetch('/foods/api/cart-count/')
           .then(response => response.json())
           .then(data => {
               document.getElementById('cart-count').textContent = data.count;
           });
   }
   ```
   
   Updates:
   ✓ On page load
   ✓ Can be called after adding/removing items
   ✓ Real-time without page refresh

3. CUSTOMIZATION STORAGE
   
   Customized items saved as JSON:
   ```json
   {
       "size": "LARGE",
       "base": "Wheat",
       "sauce": "Chili",
       "toppings": ["Onions", "Peppers", "Olives"]
   }
   ```
   
   Stored in OrderItem.customization_details field

4. ORDER STATUS WORKFLOW
   
   Default: PENDING (when order created)
   
   States:
   PENDING     → Initial state
   PROCESSING  → Order being prepared
   COMPLETED   → Order delivered/completed
   CANCELLED   → Order cancelled
   
   Can be changed from admin panel

                       COMMON ISSUES & FIXES
================================================================================

ISSUE 1: Buy button not appearing
   CAUSE: Cart is empty OR items not fetched
   FIX: 
   - Add items to cart first
   - Verify cart.html template loaded correctly
   - Check browser console for errors

ISSUE 2: Modal not opening
   CAUSE: Bootstrap JS not loaded OR jQuery issue
   FIX:
   - Verify Bootstrap JS script tag in base.html
   - Ensure jQuery or pure JS Bootstrap version
   - Check browser console for JS errors

ISSUE 3: Order not saving
   CAUSE: Migrations not run OR Order model issues
   FIX:
   - Run: python manage.py makemigrations
   - Run: python manage.py migrate
   - Check database tables created

ISSUE 4: Order ID not unique
   CAUSE: UUID generation issue
   FIX:
   - Verify uuid module imported
   - Check Order.save() method
   - Try creating new order

ISSUE 5: Cart items not clearing
   CAUSE: Delete query not executing
   FIX:
   - Verify Cart.objects.all().delete() called
   - Check database permissions
   - Debug buy_now() view

ISSUE 6: Cart count not updating
   CAUSE: API endpoint not working
   FIX:
   - Verify cart_count() view exists
   - Check URL route added
   - Test endpoint: /foods/api/cart-count/
   - Check browser fetch call

ISSUE 7: Customization details not showing
