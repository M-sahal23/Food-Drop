# CHECKOUT & PAYMENT - QUICK REFERENCE GUIDE

## 🎯 MAIN FLOW

```
Cart → Checkout (Modal) → Order Confirmation → Payment → Success → History/Tracking
```

---

## 📋 CHECKOUT PROCESS

### 1. Checkout Modal (cart.html)
- **Trigger**: Click "Proceed to Checkout" button
- **Action**: Opens modal dialog
- **Input**: Customer full name
- **Button**: "Complete Order"
- **Submits to**: `/foods/buy-now/` (POST)

### 2. Buy Now View (buy_now function)
```python
✓ Get all cart items (regular + customized)
✓ Calculate total price
✓ Create Order (auto-generates order_id)
✓ Create OrderItems for each item
✓ Clear Cart & CustomizedCart
✓ Redirect to order_confirmation page
```

**Order Status**: PENDING

### 3. Order Confirmation Page
- **URL**: `/foods/order-confirmation/<order_id>/`
- **Shows**: Order ID, Customer name, Items, Total, Status
- **Buttons**:
  - 💳 Proceed to Payment
  - 📋 View Order Details
  - 🛒 Continue Shopping

---

## 💳 PAYMENT PROCESS

### 1. Payment Page
- **URL**: `/foods/payment/<order_id>/`
- **Shows**: Order summary, Payment methods
- **Payment Methods**:
  - Credit/Debit Card
  - UPI
  - Net Banking
  - Digital Wallet
  - Cash on Delivery

### 2. Payment Methods Details

#### Credit/Debit Card
```
Inputs:
- Cardholder Name
- Card Number (16 digits) → Formatted with spaces
- Expiry Date (MM/YY) → Auto-formatted
- CVV (3 digits)
```

#### UPI
```
Inputs:
- UPI ID (yourname@googleplay)
```

#### Other Methods
```
- Net Banking: Select bank dropdown
- Wallet: Shows available balance
- COD: No payment form needed
```

### 3. Process Payment
- **URL**: `/foods/process-payment/<order_id>/` (POST)
- **In current version**: Simulated (no real payment gateway)
- **Actions**:
  - Get payment method from form
  - Update order.status = 'PROCESSING'
  - Save order
  - Redirect to payment_success

**Integration ready for**:
- Razorpay
- Stripe
- PayPal
- Others

### 4. Payment Success Page
- **URL**: `/foods/payment-success/<order_id>/`
- **Shows**:
  - ✓ Success confirmation with animation
  - Order ID
  - Order details
  - Items summary
  - Total amount paid
- **Important**: Cart is automatically cleared
- **Order Status**: PROCESSING
- **Buttons**:
  - 📍 Track Your Order
  - 📋 View Order History
  - 🛒 Continue Shopping

---

## 📍 ORDER TRACKING

### Order Tracking Page
- **URL**: `/foods/order-tracking/<order_id>/`
- **Shows**:
  
#### Timeline (3 Stages)
```
1. ✓ Order Placed (Completed)
   └─ Order confirmed and received
   
2. ⚙️ Processing (Status-dependent)
   └─ Order being prepared
   
3. 📦 Delivered (Only if COMPLETED)
   └─ Order delivered
```

#### Details
```
- Order ID
- Customer Name
- Current Status (with icon)
- Order Date & Time
- Total Items
- Items with quantities & prices
- Total Amount
- Estimated Delivery: 30-45 minutes
```

## 🗄️ DATABASE MODELS

### Order Model
```python
Fields:
- order_id (CharField, unique, auto-generated)
- customer_name (CharField)
- total_price (DecimalField)
- status (CharField: PENDING/PROCESSING/COMPLETED/CANCELLED)
- created_at (DateTimeField, auto)
- updated_at (DateTimeField, auto)

Order ID Format: ORD_TIMESTAMP_UNIQUEID
Example: ORD_202401291500_A5F3B2E9
```

### OrderItem Model
```python
Fields:
- order (ForeignKey to Order)
- food (ForeignKey to FoodItems)
- quantity (IntegerField)
- price (DecimalField, frozen at order time)
- is_customized (BooleanField)
- customization_details (TextField, JSON)

Stores individual items in each order
```

## 🔗 URL MAPPING

| URL | Method | View | Purpose |
|-----|--------|------|---------|
| `/foods/buy-now/` | POST | buy_now | Create order |
| `/foods/order-confirmation/<id>/` | GET | order_confirmation | Confirm order |
| `/foods/payment/<id>/` | GET | payment_page | Show payment form |
| `/foods/process-payment/<id>/` | POST | process_payment | Process payment |
| `/foods/payment-success/<id>/` | GET | payment_success | Success confirmation |
| `/foods/order-history/` | GET | order_history | All past orders |
| `/foods/order-tracking/<id>/` | GET | order_tracking | Real-time tracking |
| `/foods/order-status/<id>/` | GET | order_status | Order details |

## 🔄 ORDER STATUS FLOW

```
Order Created
     ↓
   PENDING ←── (User hasn't paid)
     ↓
  Payment Complete
     ↓
  PROCESSING ←── (Order being prepared)
     ↓
  Prepared & Dispatched
     ↓
  COMPLETED ←── (Order delivered)
```

**Or**:
```
At any stage:
     ↓
  CANCELLED ←── (Order cancelled/refund)
```
