3. TECHNICAL SPECIFICATIONS
═══════════════════════════════════════════════════════════════════════════════

URL ROUTING:
  Pattern:   path('<int:id>', views.Food_Details, name='fooddetails')
  Example:   /foods/5/ → Shows details for food with id=5
  Template:  {% url 'fooddetails' food.id %}
  Status:    ✓ Already configured in urls.py

FORM SUBMISSION:
  Method:    POST
  Endpoint:  addtocart (path: /foods/cart/<int:id>)
  Parameter: quantity (hidden input field)
  Behavior:  Adds/updates cart item and redirects to cart

DATABASE MODELS USED:
  • FoodItems: Main food product model
  • Cart: Regular cart items
  • CustomizedCart: Customized food items
  • SizeChart: Size options for customization
  • BaseType: Base options for customization
  • Sauce: Sauce options for customization
  • Toppings: Topping options for customization

JavaScript Functions:
  • increaseQty(): Increments quantity counter
  • decreaseQty(): Decrements quantity counter (min: 1)
  • updateQuantityField(): Syncs visible to hidden quantity field
  • Input validation: Prevents non-numeric input

4. USER WORKFLOWS
═══════════════════════════════════════════════════════════════════════════════

WORKFLOW 1: ADD TO CART WITH CUSTOM QUANTITY
─────────────────────────────────────────────
1. User visits Home page
2. Sees food items in "Top Rated" or "Popular" sections
3. Clicks on food card (image, name, or details area)
4. Navigates to Food Details page showing:
   - Large product image
   - Product name and category
   - Rating and price
   - Description
   - Quantity selector (default: 1)
   - Action buttons (Add to Cart, Customize)
5. User adjusts quantity to desired amount (e.g., 3)
6. Clicks "Add to Cart" button
7. Item added to cart with quantity 3
8. Redirected to cart page
9. User can continue shopping or proceed to checkout

WORKFLOW 2: CUSTOMIZE PRODUCT
──────────────────────────────
1. User on Food Details page
2. Clicks "Customize" button
3. Redirected to Customization page with options:
   - Size selection (if applicable)
   - Base selection (if applicable)
   - Sauce selection (if applicable)
   - Toppings selection (if applicable)
   - Quantity selector
4. User selects desired customizations
5. Sets quantity
6. Clicks "Save to Cart"
7. Customized item added to CustomizedCart
8. Redirected to cart showing customized item

WORKFLOW 3: EXPLORE FROM ALL FOODS PAGE
─────────────────────────────────────────
1. User visits All Foods page
2. Sees all available food items in grid
3. Can filter by category (Pizza, Burger, Fries, etc.)
4. Clicks on food card
5. Navigates to Food Details page
6. Can then Add to Cart or Customize
