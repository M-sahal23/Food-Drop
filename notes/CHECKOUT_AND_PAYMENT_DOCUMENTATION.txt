## 1. SYSTEM ARCHITECTURE OVERVIEW

### High-Level Flow:
```
Cart Page → Order Confirmation → Payment Page → Payment Success
    ↓              ↓                  ↓              ↓
(User Review) (Order Created) (Process Payment) (Clear Cart)
                                                  ↓
                            Order History & Tracking Available
```

### Key Components:
- **Cart Management**: Regular and customized items
- **Order Creation**: Generates unique order ID
- **Payment Processing**: Multiple payment methods
- **Order Tracking**: Real-time status updates
- **Order History**: View all past orders

### Step 3: Order Creation (buy_now view)
**File**: `foods/views.py` - `buy_now()` function
**Route**: `POST /foods/buy-now/`

**Process**:
1. Validate customer name
2. Get all cart items (regular + customized)
3. Calculate total price
4. Create Order object with auto-generated order_id
5. Create OrderItem entries for each cart item
6. Clear cart (Cart.objects.all().delete())
7. Redirect to order confirmation page

**Key Code**:
```python
def buy_now(request):
    if request.method == 'POST':
        customer_name = request.POST.get('customer_name', '').strip()
        
        # Get cart items
        regular_cart_items = list(Cart.objects.all())
        customized_cart_items = list(CustomizedCart.objects.all())
        
        # Calculate total
        grand_total = sum(...) + sum(...)
        
        # Create Order
        order = Order.objects.create(
            customer_name=customer_name,
            total_price=grand_total,
            status='PENDING'
        )
        
        # Create OrderItems from cart
        for item in regular_cart_items:
            OrderItem.objects.create(
                order=order,
                food=item.food,
                quantity=item.quantity,
                price=item.total_price,
                is_customized=False
            )
        
        # Clear cart
        Cart.objects.all().delete()
        CustomizedCart.objects.all().delete()
        
        return redirect('order_confirmation', order_id=order.id)
```

# Example: Razorpay Integration
import razorpay

def process_payment(request, order_id):
    order = Order.objects.get(id=order_id)
    
    client = razorpay.Client(auth=(RAZORPAY_KEY, RAZORPAY_SECRET))
    
    payment = client.payment.create({
        'amount': int(order.total_price * 100),
        'currency': 'INR',
        'receipt': order.order_id,
        'description': f'Order {order.order_id}'
    })

        order.status = 'PROCESSING'
        order.save()

## 5. DATABASE MODELS

### Order Model
**File**: `foods/models.py`

```python
class Order(models.Model):
    STATUS_CHOICES = [
        ('PENDING', 'Pending'),
        ('PROCESSING', 'Processing'),
        ('COMPLETED', 'Completed'),
        ('CANCELLED', 'Cancelled'),
    ]
    
    order_id = models.CharField(max_length=100, unique=True, editable=False)
    customer_name = models.CharField(max_length=100)
    total_price = models.DecimalField(max_digits=10, decimal_places=2)
    status = models.CharField(max_length=20, choices=STATUS_CHOICES, default='PENDING')
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)
    
    def save(self, *args, **kwargs):
        if not self.order_id:
            self.order_id = f"ORD_{self.created_at.strftime('%Y%m%d%H%M%S')}_{uuid.uuid4().hex[:8].upper()}"
        super().save(*args, **kwargs)
```

**Fields Explanation**:
- `order_id`: Unique identifier (auto-generated, format: ORD_TIMESTAMP_UNIQUEID)
- `customer_name`: Customer's full name
- `total_price`: Total order amount (Decimal)
- `status`: Order status (4 stages)
- `created_at`: Order creation timestamp
- `updated_at`: Last update timestamp

### OrderItem Model
**File**: `foods/models.py`

```python
class OrderItem(models.Model):
    order = models.ForeignKey(Order, related_name='items', on_delete=models.CASCADE)
    food = models.ForeignKey('FoodItems', on_delete=models.SET_NULL, null=True)
    quantity = models.IntegerField()
    price = models.DecimalField(max_digits=10, decimal_places=2)
    is_customized = models.BooleanField(default=False)
    customization_details = models.TextField(blank=True)  # JSON or text details
```

**Fields Explanation**:
- `order`: Foreign key to Order
- `food`: Reference to food item
- `quantity`: Number of items ordered
- `price`: Price at time of order (frozen)
- `is_customized`: Boolean flag for customized items
- `customization_details`: JSON string storing customization info

## 10. TESTING GUIDE

### Unit Tests:

```python
# tests.py example

from django.test import TestCase, Client
from foods.models import FoodItems, Cart, Order, OrderItem

class CheckoutPaymentTestCase(TestCase):
    
    def setUp(self):
        self.client = Client()
        self.food = FoodItems.objects.create(
            name='Pizza',
            price=250,
            rating=5,
            desc='Delicious pizza'
        )
    
    def test_buy_now_creates_order(self):
        # Create cart item
        cart = Cart.objects.create(
            food=self.food,
            quantity=2,
            total_price=500
        )
        
        # Post to buy_now
        response = self.client.post('/foods/buy-now/', {
            'customer_name': 'John Doe'
        })
        
        # Check order created
        self.assertEqual(Order.objects.count(), 1)
        order = Order.objects.first()
        self.assertEqual(order.customer_name, 'John Doe')
        self.assertEqual(order.total_price, 500)
        self.assertEqual(order.status, 'PENDING')
        
        # Check cart cleared
        self.assertEqual(Cart.objects.count(), 0)
        
        # Check redirect
        self.assertRedirects(response, f'/foods/order-confirmation/{order.id}/')
    
    def test_payment_page_loads(self):
        order = Order.objects.create(
            customer_name='John Doe',
            total_price=250,
            status='PENDING'
        )
        
        response = self.client.get(f'/foods/payment/{order.id}/')
        self.assertEqual(response.status_code, 200)
        self.assertIn('order', response.context)
    
    def test_process_payment_updates_status(self):
        order = Order.objects.create(
            customer_name='John Doe',
            total_price=250,
            status='PENDING'
        )
        
        response = self.client.post(f'/foods/process-payment/{order.id}/', {
            'payment_method': 'CARD'
        })
        
        order.refresh_from_db()
        self.assertEqual(order.status, 'PROCESSING')
```

**Implementation**:

import stripe
from django.conf import settings

stripe.api_key = settings.STRIPE_SECRET_KEY

def process_payment(request, order_id):
    order = Order.objects.get(id=order_id)
    
    session = stripe.checkout.Session.create(
        payment_method_types=['card'],
        line_items=[{
            'price_data': {
                'currency': 'inr',
                'product_data': {
                    'name': order.order_id,
                },
                'unit_amount': int(order.total_price * 100),
            },
            'quantity': 1,
        }],
        mode='payment',
        success_url=request.build_absolute_uri(
            reverse('payment_success', args=[order.id])
        ),
        cancel_url=request.build_absolute_uri(
            reverse('payment_page', args=[order.id])
        ),
    )
    
    return redirect(session.url)

## QUICK REFERENCE

### Key Models:
- **Order**: Main order record with ID, customer name, total, status
- **OrderItem**: Individual items in an order with customization details

### Key Views:
- `buy_now`: Create order from cart
- `payment_page`: Show payment form
- `process_payment`: Handle payment processing
- `payment_success`: Confirm successful payment
- `order_tracking`: Show real-time status
- `order_history`: Show all past orders

### Key Routes:
- POST `/foods/buy-now/` → Create order
- GET `/foods/payment/<id>/` → Payment form
- POST `/foods/process-payment/<id>/` → Process payment
- GET `/foods/payment-success/<id>/` → Success page
- GET `/foods/order-tracking/<id>/` → Tracking page
- GET `/foods/order-history/` → All orders

### Status Values:
- `PENDING`: Order created, waiting for payment
- `PROCESSING`: Payment successful, order being prepared
- `COMPLETED`: Order delivered
- `CANCELLED`: Order cancelled

## SUPPORT & TROUBLESHOOTING

### Common Issues:

**Q: Order ID not generating**
A: Check UUID import in models.py, ensure save() method called

**Q: Cart not clearing after payment**
A: Verify payment_success view calls Cart.objects.all().delete()

**Q: Payment page shows empty**
A: Check order_id exists in database, verify URL parameters

**Q: Timeline not showing correctly**
A: Verify order.status values match STATUS_CHOICES
