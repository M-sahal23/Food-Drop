from foods.models import Order, OrderItem

class OrderItemInline(admin.TabularInline):
    model = OrderItem
    readonly_fields = ('order', 'food', 'quantity', 'price', 'is_customized', 'customization_details')
    extra = 0
    can_delete = False

class OrderAdmin(admin.ModelAdmin):
    list_display = ('order_id', 'customer_name', 'total_price', 'status', 'created_at')
    list_filter = ('status', 'created_at')
    search_fields = ('order_id', 'customer_name')
    inlines = [OrderItemInline]

admin.site.register(Order, OrderAdmin)
admin.site.register(OrderItem)

================================================================================
                         🔑 KEY FILES EXPLAINED
================================================================================

1. models.py - WHERE DATA IS STORED
   Order model: Stores order info (ID, customer, price, status, dates)
   OrderItem model: Stores items in order (food, qty, price, customization)

2. views.py - WHERE LOGIC HAPPENS
   buy_now(): Gets cart items → Creates order → Clears cart
   order_confirmation(): Gets order → Shows confirmation page
   cart_count(): Returns JSON with item count

3. urls.py - HOW TO ACCESS VIEWS
   /buy-now/ → buy_now() view
   /order-confirmation/<id>/ → order_confirmation() view
   /api/cart-count/ → cart_count() view

4. cart.html - BUY BUTTON LOCATION
   Green button with shopping bag icon 🛍️
   Opens modal to get customer name

5. order_confirmation.html - CONFIRMATION PAGE
   Shows order ID (highlighted)
   Shows customer name
   Lists all items
   Shows total price

6. base.html - CART ICON
   Navigation bar
   Shows count badge
   Links to cart page

================================================================================
                      🔍 HOW TO DEBUG
================================================================================

If Buy Button Not Working:
1. Check browser console (F12) for errors
2. Check Django console for error messages
3. Verify cart.html has modal code
4. Verify base.html has Bootstrap CSS/JS

If Order Not Saving:
1. Check migrations ran: python manage.py migrate
2. Check database tables exist: python manage.py dbshell
3. Check for exceptions in Django console

If Cart Count Not Updating:
1. Check cart_count() view exists
2. Check /foods/api/cart-count/ URL works
3. Check browser fetch call in console
4. Verify cart.html JavaScript isn't erroring

If Orders Not Showing in Admin:
1. Verify Order registered in admin.py
2. Verify migrations applied
3. Check superuser exists
4. Reload admin page
