⚙️ BACKEND FLOW
═══════════════════════════════════════════════════════════════════════════════

1. VIEW: Food_Details (views.py, line 12-15)
   Input:  URL parameter 'id'
   Process: Fetches FoodItems object by ID
   Output: Renders fooddetails.html with fooditem context
   
2. VIEW: AddToCart (views.py, line 52-72)
   Input:  URL parameter 'id' + POST/GET 'quantity' parameter
   Process:
     - Gets FoodItems object
     - Extracts quantity (default: 1)
     - Creates or retrieves Cart object
     - Updates quantity if item exists
     - Recalculates total_price
   Output: Redirects to cart page
   
3. VIEW: Customize_Food (views.py, line 156-210)
   Input:  URL parameter 'id'
   Process:
     - Fetches food and all options
     - Determines which options to show based on category
     - Prepares context with filtered options
   Output: Renders customize.html

🐛 DEBUGGING TIPS
═══════════════════════════════════════════════════════════════════════════════

Issue: Food item not displaying
→ Check: 1) Food exists in database
         2) URL pattern correct (path('<int:id>', ...))
         3) View returns proper context

Issue: Image not showing
→ Check: 1) Image file path correct
         2) Static files collected (python manage.py collectstatic)
         3) Media settings configured in settings.py
         4) Image uploaded via admin

Issue: Add to cart not working
→ Check: 1) Quantity field has correct name (quantity-hidden)
         2) Form submits to correct URL (addtocart)
         3) AddToCart view receives quantity parameter
         4) Database Cart table exists

Issue: Customize button not working
→ Check: 1) customize URL pattern exists
         2) Customize_Food view defined
         3) customize.html template exists
