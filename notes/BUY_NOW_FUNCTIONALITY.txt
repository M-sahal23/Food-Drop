1. OVERVIEW
================================================================================

This feature enables customers to complete their purchase by clicking a "Buy Now"
button in the shopping cart. The system:

✓ Collects customer name
✓ Generates a unique Order ID with format: ORD_YYYYMMDDHHmmss_XXXXXXXX
✓ Creates an Order record with customer details and total price
✓ Stores individual items with food details and quantities
✓ Supports both regular and customized items
✓ Displays an order confirmation page
✓ Clears the cart after successful purchase

2. NEW MODELS
================================================================================

A. ORDER MODEL
   File: foods/models.py
   
   Fields:
   - order_id (CharField): Unique identifier with auto-generation
   - customer_name (CharField): Name of the customer purchasing
   - total_price (DecimalField): Total amount for the order
   - status (CharField): Order status (PENDING, PROCESSING, COMPLETED, CANCELLED)
   - created_at (DateTimeField): Timestamp when order was created
   - updated_at (DateTimeField): Timestamp when order was last updated

   Methods:
   - save(): Automatically generates unique order_id before saving
   - __str__(): Returns "Order {order_id} - {customer_name}"

   Order ID Generation:
   Format: ORD_{TIMESTAMP}_{UUID}
   Example: ORD_20260128143025_A7F2B3E1
   
   Components:
   • ORD_ = Prefix (constant)
   • TIMESTAMP = YYYYMMDDHHmmss (14 digits)
   • UUID = 8 character hex string (uniqueness guarantee)

B. ORDER ITEM MODEL
   File: foods/models.py
   
   Fields:
   - order (ForeignKey): References the Order model
   - food (ForeignKey): References the FoodItems model
   - quantity (IntegerField): Number of items ordered
   - price (DecimalField): Price of item at time of order
   - is_customized (BooleanField): Flag for customized items
   - customization_details (TextField): JSON string of customizations

   Methods:
   - __str__(): Returns "{food_name} x{quantity} in {order_id}"

5. BACKEND FUNCTIONALITY
================================================================================

A. VIEWS (foods/views.py)

   1. buy_now(request)
      
      Method: POST
      URL: /foods/buy-now/
      
      Process:
      1. Get customer name from form data
      2. Fetch all cart items (regular + customized)
      3. Calculate total price
      4. Validate cart is not empty
      5. Create Order object
      6. Create OrderItem for each regular cart item
      7. Create OrderItem for each customized cart item
      8. Serialize customization details to JSON
      9. Delete all cart items
      10. Redirect to order confirmation page

      Code Flow:
      ```
      POST /buy-now/
        ↓
      Extract customer_name
        ↓
      Get Cart.objects.all() → regular_items
      Get CustomizedCart.objects.all() → customized_items
        ↓
      Calculate grand_total
        ↓
      Validate: cart not empty
        ↓
      Order.objects.create(
          customer_name=...,
          total_price=grand_total,
          status='PENDING'
      )
        ↓
      For each item → OrderItem.objects.create()
        ↓
      Cart.objects.all().delete()
      CustomizedCart.objects.all().delete()
        ↓
      Redirect → order_confirmation
      ```

   2. order_confirmation(request, order_id)
      
      Method: GET
      URL: /foods/order-confirmation/<order_id>/
      Template: foods/order_confirmation.html
      
      Context:
      - order: Order object
      - order_items: QuerySet of OrderItem objects
      
      Process:
      1. Fetch Order by ID
      2. Get all items for this order
      3. Render confirmation template with data

B. URL ROUTING (foods/urls.py)

   New Routes:
   
   path('buy-now/', views.buy_now, name='buy_now')
   → POST form endpoint
   → Processes purchase
   
   path('order-confirmation/<int:order_id>/', 
        views.order_confirmation, 
        name='order_confirmation')
   → GET confirmation page
   → Shows order details

C. IMPORTS

   Added to foods/views.py:
   - from foods.models import Order, OrderItem
   - import json (for serializing customization details)

10. API DETAILS
================================================================================

ENDPOINT 1: Submit Order
   URL: /foods/buy-now/
   Method: POST
   Content-Type: application/x-www-form-urlencoded
   
   Parameters:
   - customer_name (required): String, max 100 chars
   - csrftoken (required): Django CSRF token
   
   Example Request:
   POST /foods/buy-now/ HTTP/1.1
   Content-Type: application/x-www-form-urlencoded
   
   customer_name=John+Doe&csrfmiddlewaretoken=xyz123
   
   Success Response: 302 Redirect
   Location: /foods/order-confirmation/5/
   
   Response Data (on confirmation page):
   - order_id: Unique order identifier
   - customer_name: Name provided
   - total_price: Calculated total
   - items: List of OrderItem objects

ENDPOINT 2: View Order Confirmation
   URL: /foods/order-confirmation/<order_id>/
   Method: GET
   
   Parameters:
   - order_id (path): Integer, Order primary key
   
   Response: HTML page with order details
   
   Context Variables:
   {
       'order': <Order object>,
       'order_items': <QuerySet of OrderItem>
   }

                           ADMIN CONFIGURATION
================================================================================

To view Orders in Django Admin, add to foods/admin.py:

   from foods.models import Order, OrderItem
   
   class OrderItemInline(admin.TabularInline):
       model = OrderItem
       readonly_fields = ('order', 'food', 'quantity', 'price', 'is_customized')
   
   class OrderAdmin(admin.ModelAdmin):
       list_display = ('order_id', 'customer_name', 'total_price', 'status', 'created_at')
       list_filter = ('status', 'created_at')
       search_fields = ('order_id', 'customer_name')
       readonly_fields = ('order_id', 'created_at', 'updated_at')
       inlines = [OrderItemInline]
   
   admin.site.register(Order, OrderAdmin)
   admin.site.register(OrderItem)
