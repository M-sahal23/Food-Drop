================================================================================
                        📊 DATA STORED AFTER ORDER
================================================================================

UNIQUE ORDER ID:
✓ Format: ORD_YYYYMMDDHHMMSS_XXXXXXXX
✓ Example: ORD_20260128143025_A7F2B3E1
✓ Auto-generated on Order creation
✓ Guaranteed unique with UUID

CUSTOMER NAME:
✓ Collected from modal input
✓ Required field
✓ Stored in Order.customer_name

FOOD ITEM NAMES:
✓ From food_name field
✓ Stored via OrderItem.food FK
✓ One record per item in order

QUANTITY:
✓ Individual per item
✓ Stored in OrderItem.quantity
✓ Can be different for each item

TOTAL PRICE:
✓ Sum of all items
✓ Stored in Order.total_price
✓ Calculated at order time

ORDER DATE:
✓ Timestamp when order created
✓ Stored in Order.created_at
✓ Format: YYYY-MM-DD HH:MM:SS.ffffff

ORDER STATUS:
✓ Default: PENDING
✓ Changeable: PROCESSING, COMPLETED, CANCELLED
✓ Stored in Order.status
✓ Can be updated from admin

CUSTOMIZATION DETAILS:
✓ For customized items
✓ Stored as JSON string
✓ Includes: size, base, sauce, toppings
✓ Stored in OrderItem.customization_details
